# Page Blueprints

## Home

Order:

1. plain-language explanation of the registry
2. systems cards
3. maturity and trust legend
4. browse-by entry points
5. source-of-truth guidance for agents and advanced users
6. registry-internals link

## Systems Index

Each card should show:

- display title
- stable seed ID
- one-sentence purpose
- lifecycle summary
- governed things count
- actions count
- read models count
- contract links

## System Detail

Each page should answer in this order:

1. what this software system is for
2. what it governs
3. how mature and authoritative it is
4. what actions it permits
5. how those actions are seen
6. which contracts and schemas define it
7. which authoritative routes ground this page

## Governed Thing Detail

Each page should answer:

1. what this thing is
2. which system owns it
3. what actions affect it
4. which read models expose it
5. which schema and contract govern it
6. which authoritative routes ground this page

## Action Detail

Each page should answer:

1. what this action does
2. who can invoke it
3. which governed thing it affects
4. which read model shows the result
5. which contract defines it
6. which authoritative routes ground this page

## Read Model Detail

Each page should answer:

1. what question this read model answers
2. which system owns it
3. which governed things it reflects
4. how fresh it is
5. which action or contract it relates to
6. which authoritative routes ground this page

## Contract Detail

Each page should answer:

1. what behavior and meaning this contract governs
2. which system it belongs to
3. which governed things it defines
4. which actions and read models it exposes
5. what its lifecycle and surface kind are
6. which authoritative routes ground this page
