# Validation

Validation evidence for this realization should cover:

- no mutation controls and no mutating browser-facing routes
- no kernel or cross-seed changes required for the refinement itself
- every reading-room grouping maps back to current authoritative registry
  routes
- canonical and permalink URLs are emitted for browsable detail resources and
  preview-only `__runs` URLs are never presented as permanent references
- systems are the default top-level entry point instead of ontology buckets
- summaries and relationships appear before raw route strings and stable IDs
- lifecycle, availability, and surface kind are presented as distinct fields
- registry-internal meta resources are clearly separated from domain software

Seed-level requirements remain in `../../acceptance.md`.
