# UI Specification

The UI should emphasize that the system manages **flows of time information**.

## Signal Board

Home screen shows connected calendars as nodes.

SOURCE CALENDARS → SYNC HUB → DESTINATION CALENDARS

Each node displays sync health.

## Event Provenance

Each event shows its origin and propagation targets.

## Propagation Graph

Events display a lineage graph showing how the event propagates.

## Timeline Mode

Horizontal timeline across all calendars.

## Conflict Visualization

Overlapping events highlighted visually.

## Operational Filters

Examples:

- conflicts
- unsynced events
- upstream changes
- propagation failures

## Event Lineage

Users can inspect the lifecycle of an event:

Origin → Transformation → Propagation → Subscribers
