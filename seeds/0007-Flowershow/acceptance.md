# Acceptance

The system is complete when:

## Core Functionality
- Shows can be created and browsed
- Entries are captured with media and placements
- Entries link to people and categories
- Judges can be associated with shows

## Taxonomy
- Entries can be tagged with taxons
- Cross-link navigation works (e.g. click "rose")

## Awards
- Awards can be defined per organization
- Awards correctly compute results

## Leaderboards
- Current season leaderboard visible per organization

## Ingestion
- Data can be ingested via API from unstructured sources (PDFs)

## Privacy
- Public view shows initials only
- Private identity mapping retained
- Suppression hides entries without deletion

---

## Nice-to-Have (not required for acceptance)
- Cross-organization leaderboards
- Advanced filtering UI
- Historical season browsing
- Flower browsing
- Category browsing
- Scientific name browsing
- Judge browsing
- Club browsing
- Individual history for a person including all media associated
- Media upload is mobile optimized and fast (with client size resize of videos and photos)
- Meida is uploaded to S3 or DO spaces
- Multiple admin operators at a show can work at the same time (adding categories, adding entries, documenting rankings, judge comments
- show summary shows winners at the top right away in real time build
