Contribution

Public projections are anonymous.

Private workspaces and mutating commands accept an admin session, the deployment service token, or an account-issued agent token with the required permissions.

Generate scoped agent tokens from /account.

Show authoring requires the "Manage shows" scope. If a command returns permission_denied, the error body names the missing scope — request a new token with that scope before retrying.

Show Authoring Workflow

To author a complete show schedule, issue commands in this order. Each step requires the id returned by the previous step.

POST {{$.BasePath}}/v1/commands/0007-Flowershow/shows.create
  { organization_id, name, location, date, season }
  => returns show with id

POST {{$.BasePath}}/v1/commands/0007-Flowershow/schedules.upsert
  { show_id }
  => returns schedule with id (use as show_schedule_id)

POST {{$.BasePath}}/v1/commands/0007-Flowershow/divisions.create
  { show_schedule_id, title, domain, sort_order, code? }
  domain: horticulture | design | special | other
  => returns division with id

POST {{$.BasePath}}/v1/commands/0007-Flowershow/sections.create
  { division_id, title, sort_order, code? }
  => returns section with id

POST {{$.BasePath}}/v1/commands/0007-Flowershow/classes.create
  { section_id, class_number, title, domain,
    description?, specimen_count?, unit?, schedule_notes? }
  => returns class with id

Read back the result:
GET {{$.BasePath}}/v1/projections/0007-Flowershow/shows/{id}
