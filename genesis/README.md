# Genesis Seed

`genesis/` is the canonical first seed.

Document boundaries:

- `README.md` explains the structure of the founding seed only.
- `brief.md` states the founding purpose, charter-level scope, and desired
  initial outcome.
- `design.md` explains what the founding seed establishes and what it
  leaves to later seeds.
- `approaches/` defines the named founding YAML approaches available for
  realization.
- `decision_log.md` records durable founding choices and rationale.
- `acceptance.md` defines founding acceptance criteria.
- `realizations/` records the founding compiled intent and the founding
  bootstrap artifacts.
